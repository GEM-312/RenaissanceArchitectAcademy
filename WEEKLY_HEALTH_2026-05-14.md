### 2-C. Hardcoded Spacing / Padding

Pervasive throughout all SwiftUI views. Most views partially use `Spacing.*` constants and partially use raw literals in the same file.

**Critical outlier — `CraftingRoomMapView.swift`:**
```swift

```
This is a magic offset that will break on different device heights. Use `GeometryReader` + `safeAreaInsets` instead.

**Other significant instances:**
- `WorkshopMapView.swift` — `.padding(.bottom, 100)` (reshuffle overlay), `.frame(width: 80)` × 4
- `ForestMapView.swift` — `.padding(.bottom, 80)` × 2 (floating label clearance)
- `NotebookView.swift` — `.padding(.bottom, 80)` (FAB clearance)
- `CraftingRoomMapView.swift` — `.frame(maxHeight: 520)`, `.frame(maxWidth: 380)`, `.frame(height: 70)`

**Recommendation:** The device-height magic numbers (80, 100, 420) should use `GeometryReader`/`safeAreaInsets`. The fixed frame sizes should be pulled into local computed properties or `Spacing` extensions. `Spacing.buttonV = 14` already exists — use it consistently.
